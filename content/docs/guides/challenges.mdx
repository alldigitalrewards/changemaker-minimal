# Challenge Management Guide

Learn how to create and manage innovation challenges that drive engagement and results.

## Overview

Challenges are the core of Changemaker. They provide a structured way to:
- Gather ideas and submissions from participants
- Set clear goals and deadlines
- Track progress and engagement
- Distribute rewards upon completion

## Creating a Challenge

### From the Admin Dashboard

1. Navigate to your workspace admin area: `/w/[your-workspace]/admin/challenges`
2. Click **Create Challenge**
3. Fill in the required information
4. Click **Save** or **Publish**

### Challenge Fields

#### Required Information

**Title**
- Clear, descriptive name for the challenge
- Example: "2025 Innovation Challenge" or "Customer Experience Ideas"

**Description**
- Detailed explanation of the challenge
- What you're looking for
- Why it matters
- Expected outcomes

**Requirements**
- Specific submission requirements
- Format expectations (text, video, document, etc.)
- Any constraints or guidelines
- Evaluation criteria

#### Optional Settings

**Dates**
- **Start Date**: When participants can begin enrolling
- **End Date**: Submission deadline
- Leave blank for ongoing challenges

**Status**
- **Draft**: Visible only to admins
- **Active**: Open for enrollments and submissions
- **Closed**: No new enrollments, existing participants can finish
- **Completed**: Archived, no new activity

**Rewards**
- Link to RewardStack SKUs
- Define reward amounts
- Set distribution criteria

## Challenge Lifecycle

### 1. Planning Phase

Before creating the challenge:
- Define clear objectives
- Identify your target participants
- Determine evaluation criteria
- Plan reward budget
- Set realistic timelines

### 2. Draft Phase

Create the challenge as a draft:
- Write compelling description
- Add detailed requirements
- Configure settings
- Review with team members
- Make adjustments as needed

### 3. Active Phase

After publishing:
- Monitor enrollments
- Answer participant questions
- Track submission progress
- Engage with participants
- Address any issues

### 4. Review Phase

When the deadline passes:
- Review all submissions
- Evaluate based on criteria
- Select winners or qualifying participants
- Prepare feedback
- Process rewards

### 5. Completion Phase

After review:
- Announce results
- Distribute rewards
- Share outcomes
- Gather feedback
- Archive the challenge

## Managing Enrollments

### Viewing Participants

Navigate to the challenge detail page to see:
- Total enrollment count
- List of enrolled participants
- Enrollment dates
- Submission status
- Completion status

### Enrollment Settings

Control who can join:
- **Open**: Anyone in the workspace can enroll
- **Invite-Only**: Participants need invitation links
- **Closed**: No new enrollments allowed

### Managing Individual Enrollments

For each enrolled participant:
- View submission details
- Track progress
- Issue rewards
- Remove enrollment (if needed)
- Send reminders

## Best Practices

### Writing Effective Challenge Descriptions

✅ **Do:**
- Be specific about what you're looking for
- Explain the problem or opportunity clearly
- Include examples of good submissions
- Set clear expectations
- Highlight the benefits of participating

❌ **Don't:**
- Use vague or generic language
- Make requirements too restrictive
- Forget to mention deadlines
- Overlook reward information
- Leave out evaluation criteria

### Setting Appropriate Timelines

**Short Challenges (1-2 weeks)**
- Good for: Quick ideation, feedback collection
- Best for: Engaged audiences, simple submissions
- Risk: May not get enough quality submissions

**Medium Challenges (1-3 months)**
- Good for: Innovation projects, detailed proposals
- Best for: Most use cases
- Sweet spot: Enough time without losing momentum

**Long Challenges (3+ months)**
- Good for: Complex solutions, ongoing programs
- Best for: Major initiatives, continuous improvement
- Risk: Participant dropout, losing interest

### Engaging Participants

**Before Launch:**
- Promote the challenge in advance
- Build anticipation
- Send calendar invites
- Create excitement

**During Challenge:**
- Send regular updates
- Share progress milestones
- Highlight interesting submissions
- Answer questions promptly
- Provide encouragement

**After Completion:**
- Recognize all participants
- Share winning submissions
- Explain evaluation decisions
- Distribute rewards promptly
- Request feedback

## Troubleshooting

### Low Enrollment

**Possible causes:**
- Challenge not promoted effectively
- Requirements too complex
- Timeline too short/long
- Rewards not compelling
- Unclear value proposition

**Solutions:**
- Improve promotion efforts
- Simplify requirements
- Adjust timeline
- Review reward structure
- Clarify benefits

### Low-Quality Submissions

**Possible causes:**
- Requirements not clear enough
- Insufficient guidance
- Lack of examples
- Participants rushed
- Wrong target audience

**Solutions:**
- Add more detailed requirements
- Provide submission examples
- Offer guidance during challenge
- Extend deadline if needed
- Better target audience selection

### Technical Issues

**Common problems:**
- Participants can't enroll
- Submission forms not working
- Dates not displaying correctly
- Notifications not sending

**First steps:**
1. Check challenge status (must be Active)
2. Verify dates are set correctly
3. Review enrollment settings
4. Check workspace permissions
5. Contact support if persists
